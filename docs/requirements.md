# 要件定義書 — 冒険者ギルド登録アプリ

**バージョン**: 1.1.0  
**最終更新**: 2026-04-21  
**ステータス**: リリース済み（GitHub Pages）

---

## 1. プロジェクト概要

### 1.1 目的
冒険者ギルドへのユーザー登録体験を、ドラゴンクエスト風のRPGインターフェースで提供する。ユーザーが名前・職業・自己紹介を楽しく入力でき、登録完了時に冒険者カードを発行する。

### 1.2 背景
「コードクエスト」学習課題 Quest 01 の達成物として作成。GitHub Pages で公開し、URLによるアクセスをゴールとする。

### 1.3 対象ユーザー
- PC・スマートフォンを利用するすべてのユーザー
- RPG・ゲームに親しみのあるユーザーを主なターゲットとする

### 1.4 公開URL
`https://iori-tominaga.github.io/guild-registration/`

## 3. 機能要件

### 3.1 登録フロー（5ステップウィザード）

#### Step 1: 名前入力
- ユーザーは冒険者名をテキスト入力できる
- 最大10文字まで入力可能
- 入力文字数をリアルタイムに表示する（例: `3/10`）
- 入力が空の場合、「けってい」ボタンは無効化される
- Enterキーでも次のステップへ進める

#### Step 2: 職業選択
- 以下9種の職業からひとつを選択できる
  - せんし / まほうつかい / そうりょ / とうぞく / しょうにん / ぶとうか / うたうたい / けんじゃ / その他
- ↑↓キーによるカーソル移動をサポートする
- Enterキーで選択を確定できる
- Escキーで前のステップへ戻れる
- カーソル位置の職業のステータス（力・素早・守備・魔力・回復）をプレビュー表示する
- 職業ごとにピクセルアートキャラクタースプライトを表示する

#### Step 3: 自己紹介入力
- ユーザーは自己紹介文をテキストエリアに入力できる
- 最大200文字まで入力可能
- 入力文字数をリアルタイムに表示する
- 「スキップ」ボタンで入力なしに次へ進める
- 「もどる」ボタンで前のステップへ戻れる

#### Step 4: 確認画面
- 入力した名前・職業・自己紹介をカード形式でプレビュー表示する
- 「はい」で登録を確定する
- 「いいえ」で前のステップへ戻る
- ←→キーでカーソルを切り替えられる

#### Step 5: 登録完了
- 登録完了メッセージをタイプライター演出で表示する
- 冒険者ギルドカードを発行・表示する（名前・職業・ランク・Lv・自己紹介・ステータス・登録日・ID）
- パーティクルエフェクトとフラッシュ演出を再生する
- 「もどる」ボタンでトップ（Step 1）へリセットする

### 3.2 UI・演出要件

| 要件 | 内容 |
|------|------|
| フォント | Press Start 2P（ドット文字フォント） |
| ウィンドウ枠 | ドラゴンクエスト風（濃紺背景・二重ボーダー・金アクセント） |
| 背景 | 星空（ラジアルグラデーション）+ 暗色背景 |
| タイプライター演出 | メッセージが1文字ずつ表示される |
| カーソル | 点滅するブロックカーソル（█） |
| 進捗表示 | 5ドットのプログレスインジケーター |
| サウンド | Web Audio API による8bitサウンド（カーソル音・決定音・キャンセル音・ファンファーレ） |
| 画面フラッシュ | 登録完了時にホワイトフラッシュ |
| パーティクル | 登録完了時に✨⭐💫🌟が舞い上がる |

### 3.3 レスポンシブ要件

| ブレークポイント | 対応内容 |
|-----------------|----------|
| 〜520px（スマホ） | 職業選択のステータスパネルを非表示・1カラム化 |
| 〜520px（スマホ） | input/textarea の font-size を16px（iOS自動ズーム防止） |
| 〜520px（スマホ） | ボタンの最小高さを44px（タッチターゲット確保） |
| 〜520px（スマホ） | キーボードヒントを非表示 |
| ノッチ付き端末 | safe-area-inset-top による上部余白確保 |

## 4. 非機能要件

| 区分 | 要件 |
|------|------|
| パフォーマンス | 初回表示3秒以内（Google Fonts・CDN読み込み含む） |
| ブラウザ対応 | Chrome / Safari / Firefox / Edge の最新版 |
| モバイル対応 | iOS Safari / Android Chrome で正常動作すること |
| アクセシビリティ | キーボードのみで全操作が完了できること |
| セキュリティ | 入力値はクライアント内のみで処理し、外部送信しない |
| データ保持 | データはセッション内のみ保持（永続化なし） |

---

## 5. 制約・除外事項

- バックエンド・データベースは持たない
- ユーザー認証は行わない
- 登録データのCSV出力・共有機能は対象外
- 多言語対応は対象外（日本語のみ）
