# external-spec.md 更新エージェント指示書

`/update-qa` スキルから Agent として呼ばれます。
`docs/external-spec.md` をコードの変更内容に合わせて更新し、新しいテストケースを追加してください。

## 手順

### 1. 変更内容の把握

```bash
git log --oneline -3          # 直近のコミット確認
git diff HEAD~1 HEAD          # 直前コミットの差分
```

### 2. 次のバージョン番号の決定

`docs/requirements.md` と同じロジックでバージョンを決定する（同じバージョンにする）：

| 条件 | 操作 |
|---|---|
| "Quest" を含む | X を +1、Y・Z を 0 にリセット |
| 通常の機能追加・変更 | Y を +1、Z を 0 にリセット |
| "fix" のみ | Z を +1 |

### 3. 既存の最大 TC 番号の確認

`docs/external-spec.md` 全体を読み、`TC-XX` の最大番号を確認する。
新しい TC は その番号 + 1 から採番する。

### 4. 新しいテストケースの追加

変更された機能ごとに TC を追加する。書式：

```markdown
### TC-XX: [機能名]

**目的**: [何を確認するか1文で]

| # | 操作手順 | 期待結果 |
|---|----------|----------|
| XX-1 | [操作] | [具体的な期待結果] |
```

- 期待結果は **具体的に**（「正しく動作する」ではなく「○○が表示される」）
- 操作手順は **誰でも再現できる粒度** で書く

### 5. ヘッダーと変更履歴の更新

- ヘッダーの **バージョン** と **対象バージョン** を更新する
- 「8. 変更履歴」に新しい行を追記する（書式: `| X.Y.Z | 日付 | 内容 |`）

### 6. 完了報告

追加した **TC 番号と内容の要約** をメインの Claude に返す。

## 注意事項

- **コミットしない**（コミットはメインの Claude が行う）
- 今日の日付は currentDate コンテキストから取得する
