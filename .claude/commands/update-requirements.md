# requirements.md 更新エージェント指示書

`/update-qa` スキルから Agent として呼ばれます。
`docs/requirements.md` をコードの変更内容に合わせて更新してください。

## 手順

### 1. 変更内容の把握

```bash
git log --oneline -3          # 直近のコミット確認
git diff HEAD~1 HEAD          # 直前コミットの差分
```

### 2. 現在のバージョンの読み取り

`docs/requirements.md` のヘッダー（**バージョン**: X.Y.Z）を確認する。

### 3. 次のバージョン番号の決定

コミットメッセージを参照して判断する：

| 条件 | 操作 |
|---|---|
| "Quest" を含む | X を +1、Y・Z を 0 にリセット |
| 通常の機能追加・変更 | Y を +1、Z を 0 にリセット |
| "fix" のみ | Z を +1 |

### 4. requirements.md の更新

以下を編集する：
- ヘッダーの **バージョン** と **最終更新** を更新する
- 変更に対応する機能要件（3.x 節）の記述を追加・修正する
- 「7. 変更履歴」に新しい行を追記する（書式: `| X.Y.Z | 日付 | 内容 |`）

### 5. 完了報告

更新した **バージョン番号** と **変更した箇所の要約** をメインの Claude に返す。

## 注意事項

- **コミットしない**（コミットはメインの Claude が行う）
- 今日の日付は currentDate コンテキストから取得する
